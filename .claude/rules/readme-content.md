# README Content Rules

When editing README.md:

1. **Identity = Backend Engineer** — header desc and About Me must lead with backend, not AI
2. **About Me ordering**: backend-core domains first, AI/productivity last
3. **No hardcoded career year** — "N년차" drifts every year. Use domain-focused intro instead
4. **shields.io badges use `style=flat-square`** — consistent across all badges
5. **OSS summary stats must match actual PR count** — repos/PRs/Merged in summary line = counted from table rows
6. **Date ordering is reverse-chronological** — newest first in Activities, Certifications
7. **All external links must be valid** — no broken URLs
8. **Markdown table column counts must be consistent** — header columns = data row columns
9. **Identity source of truth** — resume repo `src/content/resume/ko.json` `personalInfo.title`
